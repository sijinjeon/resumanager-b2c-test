# 🎯 강점진단 시스템 업데이트 가이드

> 이 문서는 비개발자가 이해할 수 있도록 작성된 강점진단 시스템 업데이트 안내서입니다.

---

## 📋 개요

### 현재 상태 → 업데이트 후 상태

| 항목 | 현재 | 업데이트 후 |
|------|------|-------------|
| 성향 개수 | 12개 | 12개 (동일) |
| 성향 명칭 | 영어 기반 (legacy, novelty 등) | 한글 기반 (응용, 개척 등) |
| 질문 개수 | 25개 (A vs B 비교형) | 약 120개 (성향별 10개) |
| 질문 형식 | A vs B 양자택일 | 동의 정도 (리커트 척도) |
| 결과 리포트 | 기본 설명 | Why 리포트 (건축물 비유 포함) |

## 🏗️ 12개 성향 매핑표

기존 시스템의 성향과 새로운 가이드의 성향을 매핑한 표입니다:

| 기존 영문 ID | 기존 한글명 | → | 새로운 명칭 | 새로운 ID |
|-------------|------------|---|-----------|----------|
| legacy | 기존 | → | **응용** | application |
| novelty | 새로움 | → | **개척** | pioneering |
| stability | 안정 | → | **안정** | stability |
| challenge | 도전 | → | **도전** | challenge |
| goal | 목표 | → | **달성** | achievement |
| purpose | 목적 | → | **해결** | solution |
| information | 정보 | → | **이해** | understanding |
| insight | 인사이트 | → | **확장** | expansion |
| person | 사람 | → | **상생** | coexistence |
| situation | 상황 | → | **대응** | response |
| together | 함께 | → | **협력** | cooperation |
| mySelf | 내가 | → | **증명** | proof |

## 📝 성향별 질문 문항 정리

각 성향별로 새로운 가이드에서 제공하는 질문 문항입니다:

### 1. 응용 (Application) - 10문항
```
□ 나는 전통이나 관습은 검증된 지혜라고 생각한다.
□ 나는 정도나 정석을 참고하여 정답을 맞추려 노력한다.
□ 나는 관련 자료나 기록이 있으면 이를 활용한다.
□ 검증된 방법이라면 그대로 활용할 수 있다.
□ 내가 오랫동안 해온 일에 대해 자부심을 갖는다.
□ 나는 문제를 해결할 때 레퍼런스를 찾으려 노력한다.
□ 나의 업적으로 타인에게 영감을 주고 싶다.
□ 세상에 이미 존재하는 수많은 보석을 다 모으고 싶다.
□ 경험이 많은 전문가의 도움을 받고 싶다.
□ 틀이 존재할 때 안정감을 느낀다.
```

### 2. 개척 (Pioneering) - 10문항
```
□ 나는 전통이나 관습은 딛고 넘어서야 하는 발판이라고 생각한다.
□ 나는 틀리더라도 새로운 방법을 찾거나 만들어 내려고 노력한다.
□ 나는 백지에서부터 직접 만든 결과물을 타인과 공유한다.
□ 검증된 방법이라도 더 나은 방법이 있을 것이다.
□ 새롭게 시작한 일에 대해 자부심을 갖는다.
□ 나는 문제를 해결할 때 머릿속으로 들어간다.
□ 나의 색다름으로 타인에게 영감을 주고 싶다.
□ 존재하는 줄도 몰랐던 숨겨진 보석을 찾고 싶다.
□ 새로운 시도를 하는 개척자의 도움을 받고 싶다.
□ 틀을 벗어나는 것이 불편하지 않다.
```

### 3. 안정 (Stability) - 11문항
```
□ 되돌아 봤을 때 오점을 남기기 싫다.
□ 조심하고 꼼꼼하게 처리하면 실패할 일은 없다.
□ 나는 성이 무너지지 않게 돕는 사람이다.
□ 그동안 문제 없던 방법이라도 오류가 없는지 검증해본 후 사용한다.
□ 내가 한 일은 그동안 문제가 없었다는 것에 자부심을 갖는다.
□ 나는 당면한 문제 자체를 세밀하게 분석한다.
□ 큰 성공보다 중요한 것은 큰 실패를 예방하는 것이다.
□ 후대에 물려줄 유산은 지켜낸 자산이다.
□ 나는 조심스럽다.
□ 문제 상황을 원상복구하면 마음이 편해진다.
□ 향후 문제가 될 소지를 줄일 방법을 고민한다.
```

### 4. 도전 (Challenge) - 11문항
```
□ 지난 일은 뒤로 하고 앞으로 가능한 것에 집중하고 싶다.
□ 나를 믿고 내딛은 과감한 한 발자국이 성공을 가져온다.
□ 나는 완성된 성을 머리에 그리고 벽돌 하나를 쌓는 사람이다.
□ 검증된 방법은 일단 적용하고 가면서 보완한다.
□ 문제에서 배우고 지속적으로 개선했다는 것에 자부심을 갖는다.
□ 나는 문제를 해결했을 때 얻게 될 이득을 상상한다.
□ 현실은 누구도 예측할 수 없고 리스크는 늘 존재한다.
□ 후대에 물려줄 유산은 앞으로 내딛은 한걸음이다.
□ 나는 멈춰본 적이 없다.
□ 문제 상황을 기회삼아 돌파해내고 싶다.
□ 문제를 미리 고려하기보다는 겪어 나가면서 해결하고자 한다.
```

### 5. 달성 (Achievement) - 10문항
```
□ 눈에 보이는 결과로 증명할 수 있어야 한다.
□ 끝까지 해내는 것이 중요하다.
□ 세상이 나를 평가한다.
□ 성공할 때까지 노력해야 한다.
□ 후대에 물려줄 유산은 내가 이룬 업적이다.
□ 나는 야망이 있다.
□ 나의 현재 성과에 대한 측정자료가 필요하다.
□ 모로 가도 서울만 빨리 가면 된다.
□ 나의 성과가 나의 자부심이다.
□ 내가 이루어낸 것들이 내가 어떤 사람인지 말해준다.
```

### 6. 해결 (Solution) - 10문항
```
□ 눈에 보이지 않는 고민과 노력이 퀄리티의 차이를 만든다.
□ 제대로 하는 것이 중요하다.
□ 세상으로부터 나를 지켜내야 한다.
□ 스스로 만족할 때까지 노력해야 한다.
□ 후대에 물려줄 유산은 내가 실현한 가치이다.
□ 나는 사명이 있다.
□ 나의 현재 방향에 대한 확신이 필요하다.
□ 출발 전에 어떤 방법으로 갈지 결정하는 것이 중요하다.
□ 나의 높은 기준이 나의 자부심이다.
□ 내가 추구하는 목적이 내가 어떤 사람인지 말해준다.
```

### 7. 이해 (Understanding) - 10문항
```
□ 정답은 이미 현실에 있다.
□ 아는 것이 힘이다.
□ 나는 정확한 사람이다.
□ 내가 결정하는데 필요한 자료를 확보하는 것이 중요하다.
□ 지도를 보고 확실한 길을 찾으면 출발하자.
□ 나의 정확성이 나의 자부심이다.
□ 객관적인 근거로 증명할 수 있어야 한다.
□ 나의 결론을 변호할 수 있어야 한다.
□ 유식한 사람으로 인정받고 싶다.
□ 나는 사실과 정보를 믿는다.
```

### 8. 확장 (Expansion) - 10문항
```
□ 정답은 내가 만들어 가는 것이다.
□ 지식이 서 말이라도 꿰어야 보배.
□ 나는 나만의 관점이 있는 사람이다.
□ 내가 결정하는데 필요한 생각할 시간을 갖는 것이 중요하다.
□ 주요 랜드마크와 상대적 위치가 머릿속에 그려지면 출발하자.
□ 나의 통찰력이 나의 자부심이다.
□ 검증은 각자가 스스로 해야한다.
□ 나의 관점을 변호할 수 있어야 한다.
□ 현명한 사람으로 인정받고 싶다.
□ 나는 사실과 정보를 기반으로 도출한 내 생각을 믿는다.
```

### 9. 상생 (Coexistence) - 10문항
```
□ 나는 따뜻한 사람이고 싶다.
□ 내게 소중한 사람들을 지키고 싶다.
□ 관계가 탄탄해야 일이 잘 된다.
□ 주변 사람을 불편하게 하지 않으려 노력한다.
□ 내가 가꿔온 관계에 자부심을 갖는다.
□ 나는 모두가 만족할 대안을 찾으려 노력한다.
□ 나는 의지할 곳을 제공하고 싶다.
□ 탄탄한 팀워크가 중요하다.
□ 나를 편안하게 해주는 사람이 좋다.
□ 관계를 고려하여 소통하려 노력한다.
```

### 10. 대응 (Response) - 10문항
```
□ 나는 도움이 되는 사람이고 싶다.
□ 혼란을 해결하고 싶다.
□ 일이 잘 풀려야 관계도 원활해진다.
□ 허둥대지 않고 침착하려 노력한다.
□ 내가 해결한 문제들에 자부심을 갖는다.
□ 나는 문제에 가장 적합한 답을 찾으려 노력한다.
□ 나는 해결책을 제공하고 싶다.
□ 똑똑하고 빠릿빠릿한 팀원들이 중요하다.
□ 나를 도와주는 사람에게 호감이 간다.
□ 해결을 위한 대화가 우선이다.
```

### 11. 협력 (Cooperation) - 10문항
```
□ 우리 덕분에 내가 존재할 수 있어 감사하다.
□ 뭉치면 살고 흩어지면 죽는다.
□ 나는 모두에게 인정받을 때 기쁘다.
□ 남는 건 함께한 시간이다.
□ 의견을 모으는 일은 중요하다.
□ 일은 매끄럽게 진행해야 한다.
□ 발생한 문제로 인해 누가 영향을 받는지 살핀다.
□ 후대에 물려줄 자산은 우리가 만든 공동체이다.
□ 나는 배려깊은 사람이다.
□ 공동체 내에서 서로 소통하며 맞춰 나가는 것이 중요하다.
```

### 12. 증명 (Proof) - 10문항
```
□ 내 덕분에 우리가 존재할 수 있어 자랑스럽다.
□ 난세는 영웅을 만든다.
□ 나는 모두를 설득하고 움직일 때 기쁘다.
□ 남는 건 나의 기여다.
□ 다수결이 항상 옳은 것은 아니다.
□ 쉬운 길은 없기에 각오와 집념으로 극복해야 한다.
□ 문제 해결에 나의 역할이 중요한 이유를 고민한다.
□ 후대에 물려줄 유산은 모범이 된 위인들의 이야기이다.
□ 나는 꼭 필요한 사람이다.
□ 공동체 안에서는 나를 드러내는 것도 중요하다.
```

## 🔄 업데이트 진행 단계 (Step by Step)

### STEP 1: 현재 상태 백업 ✅
> **담당**: 개발자
> **예상 시간**: 5분

- [ ] 기존 `data/questions.json` 백업
- [ ] 기존 `data/personalities.json` 백업
- [ ] 백업 파일명: `questions_backup_YYYYMMDD.json`

---

### STEP 2: 질문 데이터 구조 결정 📋
> **담당**: 기획자 (사용자님) 결정 필요
> **예상 시간**: 30분

현재 두 가지 방식 중 선택이 필요합니다:

#### 옵션 A: 기존 방식 유지 (A vs B 비교형)
```json
{
  "questionA": "나는 전통이나 관습은 검증된 지혜라고 생각한다",
  "questionB": "나는 전통이나 관습은 딛고 넘어서야 하는 발판이라고 생각한다",
  "typeA": "application",
  "typeB": "pioneering"
}
```
- **장점**: 기존 UI 수정 최소화, 상반되는 성향 명확한 구분
- **단점**: 질문 수 제한, 중립 선택 어려움

#### 옵션 B: 리커트 척도 방식 (새로운 방식)
```json
{
  "question": "나는 전통이나 관습은 검증된 지혜라고 생각한다",
  "type": "application",
  "options": ["전혀 아니다", "아니다", "보통이다", "그렇다", "매우 그렇다"]
}
```
- **장점**: 모든 질문 활용 가능, 정밀한 점수 측정
- **단점**: UI 전면 수정 필요, 개발 시간 증가

**🤔 결정 필요**: 어떤 방식으로 진행할까요?

---

### STEP 3: 새로운 데이터 파일 생성 📝
> **담당**: 개발자
> **예상 시간**: 2-4시간
