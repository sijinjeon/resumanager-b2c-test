# 📋 Why 강점 보고서 구현 가이드

이 문서는 `강점진단why 결과.md` 파일의 데이터를 활용하여 **Why 성향만** 보고서 형태로 출력하는 기능을 구현하기 위한 가이드입니다.

---

## 📌 현재 상태 vs 원하는 상태

### 🔵 현재 상태

| 항목 | 내용 |
|------|------|
| 성향 종류 | Why 6개 + How 6개 = 총 12개 |
| 결과 화면 | Why와 How 둘 다 표시 |
| 데이터 | `data/personalities.json` 사용 |
| 표시 내용 | 이름, 설명, 키워드, 강점 |

**현재 Why 성향 6개:**
- 기존(Legacy), 새로움(Novelty), 안정(Stability)
- 도전(Challenge), 목표(Goal), 목적(Purpose)

### 🟢 원하는 상태

| 항목 | 내용 |
|------|------|
| 성향 종류 | **Why 13개만** (How 제외) |
| 결과 화면 | **Why만** 표시 |
| 데이터 | `강점진단why 결과.md` 기반 새 데이터 파일 |
| 표시 내용 | 건축물 비유, Why 요약 문구, 요약 동사, 설명 문장, 설명 문단 |

**새로운 Why 성향 13개:**
1. 응용 (아쿠아덕트)
2. 개척 (구겐하임 박물관)
3. 안정 (엠파이어 스테이트)
4. 도전 (부르즈 할리파)
5. 달성 (기자 대피라미드)
6. 해결 (스발바르 종자저장고)
7. 이해 (스톤헨지)
8. 확장 (국제우주정거장)

## 📝 상세 구현 단계

### 1단계: 새 데이터 파일 생성 ⭐

**파일 위치:** `data/why-personalities.json`

**역할:** `강점진단why 결과.md`의 내용을 JSON 형식으로 변환

**데이터 구조 예시:**
```json
{
  "application": {
    "id": "application",
    "name": "응용",
    "building": {
      "name": "아쿠아덕트",
      "description": "길게는 2,000년 전에 지어진 로마시대의 아쿠아덕트는..."
    },
    "summary": "위대한 유산",
    "verbs": ["#쌓아가다", "#이어가다"],
    "shortDescription": "님은 탄탄하게 쌓아올려 무너지지 않는 결과물을 만들고 싶어합니다.",
    "fullDescription": "나의 가치는 내가 없을 때, 내가 떠난 뒤에 더 빛난다고 믿습니다..."
  }
}
```

**포함할 필드:**
| MD 파일 컬럼 | JSON 필드 | 설명 |
|-------------|-----------|------|
| 명칭 | `name` | 성향 이름 |
| 건축물 비유 | `building` | 건축물 이름 + 설명 |
| Why 요약 문구 | `summary` | 짧은 요약 문구 |
| W 요약 동사(#) | `verbs` | 해시태그 형태의 동사들 |
| W 설명 문장 | `shortDescription` | 한 줄 설명 |
| W 설명 문단 | `fullDescription` | 상세 설명 문단 |

---

### 2단계: 질문 데이터 연결 ⭐

**현재 상황:**
- 질문 파일: `data/questions.json`
- 각 질문에는 `typeA`와 `typeB`가 있어서 두 성향을 비교

**필요한 작업:**
- 13가지 새 성향에 맞게 질문의 `typeA`, `typeB` 값 매핑
- 예: `legacy` → `application`, `novelty` → `pioneer`

**매핑 테이블 (예상):**

| 현재 값 | 새 값 | 한글 명칭 |
|---------|-------|----------|
| legacy | application | 응용 |
| novelty | pioneer | 개척 |
| stability | stability | 안정 |
| challenge | challenge | 도전 |
| goal | achievement | 달성 |
| purpose | solution | 해결 |
| - | understanding | 이해 |
| - | expansion | 확장 |
| - | harmony | 상생 |
| - | response | 대응 |
| - | cooperation | 협력 |
| - | proof | 증명 |

⚠️ **참고:** 현재 6개에서 13개로 늘어나므로, 질문도 추가/수정이 필요할 수 있습니다.

## ⚠️ 중요 결정 사항

구현 전에 확인이 필요한 사항들입니다:

### 1️⃣ 성향 개수 변경

**현재:** Why 6개
**새로운:** Why 13개 (표에서 확인)

> ❓ **질문:** 기존 6개와 새로운 13개의 관계가 어떻게 되나요?
> - 완전히 새로운 체계인가요?
> - 기존 6개가 13개로 확장/분리된 것인가요?

### 2️⃣ 질문 재설계 필요

**현재 질문:** 6개 Why 성향 기반
**새 질문:** 13개 Why 성향 기반

> ❓ **질문:** 새로운 질문 세트가 준비되어 있나요?
> - 있다면: 새 질문 파일 경로를 알려주세요
> - 없다면: 13개 성향에 맞는 질문을 새로 설계해야 합니다

### 3️⃣ How 완전 제거 여부

> ❓ **질문:** How 관련 기능을 완전히 제거하나요?
> - 결과 화면에서만 숨기기
> - 또는 질문/계산/저장 모든 곳에서 제거
